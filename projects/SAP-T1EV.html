Project Description

SAP Enterprise Event Platform

Event management platform built on Adobe Experience Manager (AEM). Fixed-price project. Focused on the future state of large Tier-1 events that take place a few times a year, such as Sapphire, TechEd, etc.

Responsibilities

  • Develop and maintain:
    • multiple custom AEM page components, covering author dialogs;
    • admin tools;
    • AEM templates;
    • integrations of AEM with other third-party platforms and services;
  • Familiarity and working knowledge of common AEM authoring practices, including Content Policies, Style System, Editable Template Modes (Structure & Initial Content), and Responsive Layouts, as well as AEM i18n Translator tools and DAM topics such as Image & Video Profiles, Asset Processing, Smart Cropping, and Renditions;
  • Work closely in a cross-functional setup with FE devs on headless AEM features, delivering content via Sling Models Exporter (JSON) for ReactJS-based rendering; solid understanding of AEM component edit config listeners;
  • Adhere to internal best practices with respect to CI/CD, such as the Git Flow branching model, hotfix flows, and so on;
  • Write documentation and prepare release verification steps for manual QA;
  • Work on technical debt:
    • resolve Sonar and Checkmarx issues (refactor code, simplify code complexity and duplicated code, fix security issues such as XSS vulnerabilities, and so on);
    • update dependencies (libraries, frameworks, technology versions);
  • Collaborate with QA and BA to detect, investigate, and fix bugs;
  • Communicate with performance QA to identify and resolve bottlenecks, supporting platform health: use different optimization techniques for AEM applications including caching strategies, lazy loading, and minimizing server calls, etc.;
  • Participate in team collaboration through meetings, troubleshooting, sharing knowledge, and reviewing pull requests.

Achievements

  • Implemented a complex, highly configurable component - Sticky Section featuring dual containers with synchronized scrolling and dynamic multi-column ratios, supporting layout breakpoint behavior for nested responsive grids, including advanced template structure configuration with per-container "allowed components" policies;
  • Customized ACS AEM Commons Generic Lists;
  • Updated the Apache Solr version;
  • Participated in the platform migration to a new version of the AEM Service Pack. Identified and reported a layout-related bug in the responsivegrid component to the Adobe team;
  • Used AEM Search API to investigate affected areas related to semantic tag fixes across all Rich Text content on the platform and developed responsive Groovy-based batch processing scripts to update the content at scale;
  • Involved in the AEM cloud transformation as EDS (Edge Delivery Services), engaging in technical discovery, participating in regular tech talks and cross-team discussions, exploring documentation and existing features, etc.
