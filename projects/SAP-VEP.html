Project Description

SAP Marketo Integration with Event Platform

SAP Event Platform is part of the SAP Digital Microsite Platform, a self-service publishing stack that allows SAP Marketers to configure and self-publish customer-/partner-facing websites for events, webinars, and content hubs.

Project Role(s)

Engineer (Full-Stack Developer)

Responsibilities

  • Develop and maintain Java Spring Boot microservices, implementing business logic and REST endpoints;
  • Ensuring robust integration between internal and external services through reliable data exchange and contract consistency;
  • Understanding of microservices architectural principles, including horizontal scalability and fault tolerance strategies such as retry mechanisms, system resilience and stable performance under load;
  • Using Granite UI and Coral UI, design and implement custom Touch UI admin tools based on user story specifications, ensuring maintainability and usability, with a focus on an intuitive authoring experience;
  • Strong understanding of RESTful API design principles and working knowledge of integrating AEM with other systems through APIs;
  • Extensive experience in Granite UI and Coral UI customization for Touch UI development and on such topics as implementing complex configuration interfaces;
  • Follow internal best practices with respect to coding standards, unit test coverage, and continuous integration;
  • Troubleshoot, debug, and resolve technical issues efficiently while maintaining system stability;
  • Contribute to project documentation and support knowledge sharing across the team;
  • Actively participate in Agile processes, including daily stand-ups, provide regular progress updates, engage in team discussions and planning.

Additional Info

Achievements

  • Implemented a REST-based Spring Boot microservice from scratch to integrate two systems, including authenticated requests for secure downstream API communication and deployment via CI/CD pipelines, and integrated it with AEM, covering the UI part (error handling, front-end validation, and multi-layer backend validation);
  • Developed an AEM admin tool page with a dynamically expanding table, search/filtering panel, and the ability to select specific or all items, including validation, error handling, and data preparation for sending to the target service.
